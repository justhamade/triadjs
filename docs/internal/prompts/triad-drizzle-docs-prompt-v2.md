# Triad Documentation — Drizzle ORM Integration

## Prompt for Claude Code

You are writing documentation for **Triad**, a TypeScript/Node.js API framework where specification, implementation, validation, and testing are a single source of truth.

Triad's `t.model()` definitions are the single source of truth for the API contract AND the database schema. The `@triad/drizzle` package generates Drizzle ORM table definitions directly from Triad models, eliminating the need to maintain a separate database schema.

Create a documentation file (`docs/drizzle-integration.md`) covering the full integration story.

---

## Why Drizzle (not Prisma)

Include a brief section:

- **Same philosophy**: Both Triad and Drizzle are code-first TypeScript — no separate schema language, no code generation step for types.
- **Prisma introduces drift**: Prisma's `.prisma` schema is a second source of truth that must be manually kept in sync. `prisma generate` is a drift opportunity Triad exists to eliminate.
- **Generated, not maintained**: Triad generates the Drizzle schema from `t.model()` + `.storage()` hints. You never write `pgTable()` by hand.
- **Lightweight**: ~7.4kb, zero dependencies, serverless-ready.

Note: Triad doesn't require Drizzle. The framework is ORM-agnostic. But Drizzle is the recommended and documented choice because it shares the code-first philosophy and its schemas can be auto-generated.

## Core Concept: One Schema, Two Outputs

This is the key insight. The Triad model produces BOTH the API contract (OpenAPI, validation, types) AND the database schema (Drizzle tables, migrations):

```
t.model('Pet', { ... })
  .storage({ ... })
        │
        ├──→ triad docs     → OpenAPI 3.1 spec
        ├──→ triad gherkin   → .feature files
        ├──→ triad test      → behavior tests
        └──→ triad db        → Drizzle schema (pgTable definitions)
                                    │
                                    └──→ drizzle-kit generate → SQL migrations
```

### The Model with Storage Hints

```typescript
import { t } from '@triad/core';

const Pet = t.model('Pet', {
  id: t.string().format('uuid').identity()
    .doc('Unique pet identifier')
    .storage({ defaultRandom: true }),

  name: t.string().minLength(1).maxLength(100)
    .doc('Pet name').example('Buddy'),

  species: t.enum('dog', 'cat', 'bird', 'fish')
    .doc('Species'),

  age: t.int32().min(0).max(100)
    .doc('Age in years').example(3),

  status: t.enum('available', 'adopted', 'pending')
    .doc('Adoption status')
    .default('available'),

  tags: t.array(t.string()).optional()
    .doc('Searchable tags'),

  createdAt: t.datetime()
    .doc('Record creation timestamp')
    .storage({ defaultNow: true }),
})
.storage({
  tableName: 'pets',
  columns: {
    // Database-only columns — NOT exposed in the API
    updatedAt: { type: 'timestamp', defaultNow: true },
    deletedAt: { type: 'timestamp', nullable: true },
    internalNotes: { type: 'text', nullable: true },
    intakeSource: { type: 'varchar', length: 50, nullable: true },
  },
  indexes: [
    { columns: ['name', 'species'], unique: true },
    { columns: ['status'] },
  ],
});
```

  age: integer('age').notNull(),
  status: statusEnum('status').notNull().default('available'),
  tags: text('tags').array(),

  intakeSource: varchar('intake_source', { length: 50 }),

## Storage Hints — What They Control

### Field-level `.storage()`

```typescript
// Default generation strategies
t.string().format('uuid').storage({ defaultRandom: true })    // → uuid().defaultRandom()
t.datetime().storage({ defaultNow: true })                     // → timestamp().defaultNow()
t.int32().storage({ type: 'serial' })                         // → serial() instead of integer()

// Column name override (API field name ≠ DB column name)
t.string().storage({ column: 'full_name' })                   // field "name" → column "full_name"

// Custom DB type override
t.string().storage({ dbType: 'citext' })                      // → use citext extension type
```

### Model-level `.storage()`

```typescript
.storage({
  tableName: 'pets',             // Override table name (default: lowercase plural of model name)
  schema: 'public',             // PostgreSQL schema

  columns: {                     // Database-only columns
    updatedAt: { type: 'timestamp', defaultNow: true },
    deletedAt: { type: 'timestamp', nullable: true },
    version: { type: 'integer', default: 1 },
  },

  indexes: [
    { columns: ['name', 'species'], unique: true },
    { columns: ['status'] },
    { columns: ['createdAt'], order: 'desc' },
  ],

  softDelete: 'deletedAt',      // Shorthand: enables soft delete pattern on this column
})
```

## The Repository — Simplified by Generation

Since the Drizzle schema is generated from the Triad model, the mapping between them is predictable. Show a repository that leverages this:

```typescript
import { eq, and, isNull } from 'drizzle-orm';
import { pets } from './db/schema';  // AUTO-GENERATED by Triad

// Triad model types (API contract)
type Pet = t.infer<typeof Pet>;
type CreatePetInput = t.infer<typeof CreatePet>;

// Drizzle row types (storage contract)
type PetRow = typeof pets.$inferSelect;

export class PetRepository {
  constructor(private db: DrizzleDB) {}

  // Mapping is straightforward because both schemas come from the same source
  private toApi(row: PetRow): Pet {
    return {
      id: row.id,
      name: row.name,
      species: row.species,
      age: row.age,
      status: row.status,
      tags: row.tags ?? undefined,
      createdAt: row.createdAt.toISOString(),
    };
    // Storage-only columns (updatedAt, deletedAt, internalNotes) are excluded
  }

  async create(input: CreatePetInput): Promise<Pet> {
    const [row] = await this.db.insert(pets).values({
      name: input.name,
      species: input.species,
      age: input.age,
      tags: input.tags ?? null,
      // id, status, createdAt, updatedAt handled by DB defaults
    }).returning();
    return this.toApi(row);
  }

  async findById(id: string): Promise<Pet | null> {
    const [row] = await this.db
      .select()
      .from(pets)
      .where(and(eq(pets.id, id), isNull(pets.deletedAt)));
    return row ? this.toApi(row) : null;
  }

  async list(filters: {
    species?: string;
    status?: string;
    limit: number;
    offset: number;
  }): Promise<Pet[]> {
    const conditions = [isNull(pets.deletedAt)];
    if (filters.species) conditions.push(eq(pets.species, filters.species));
    if (filters.status) conditions.push(eq(pets.status, filters.status));

    const rows = await this.db
      .select()
      .from(pets)
      .where(and(...conditions))
      .limit(filters.limit)
      .offset(filters.offset);

    return rows.map(row => this.toApi(row));
  }

  async findByNameAndSpecies(name: string, species: string): Promise<Pet | null> {
    const [row] = await this.db
      .select()
      .from(pets)
      .where(and(
        eq(pets.name, name),
        eq(pets.species, species),
        isNull(pets.deletedAt),
      ));
    return row ? this.toApi(row) : null;
  }

  async softDelete(id: string): Promise<boolean> {
    const result = await this.db
      .update(pets)
      .set({ deletedAt: new Date() })
      .where(eq(pets.id, id));
    return result.rowCount > 0;
  }
}
```

## Relations Between Models

Show how relations are declared and generated:

```typescript
const Owner = t.model('Owner', {
  id: t.string().format('uuid').identity().storage({ defaultRandom: true }),
  name: t.string().minLength(1).doc('Owner name'),
  email: t.string().format('email').doc('Contact email'),
})
.storage({ tableName: 'owners' });

const Pet = t.model('Pet', {
  id: t.string().format('uuid').identity().storage({ defaultRandom: true }),
  name: t.string().minLength(1).doc('Pet name'),
  ownerId: t.string().format('uuid').optional()
    .doc('Owner ID')
    .storage({ references: 'owners.id' }),     // Foreign key
  // ...
})
.storage({
  tableName: 'pets',
  relations: {
    owner: { type: 'many-to-one', model: 'Owner', field: 'ownerId' },
  },
});
```

Generated Drizzle output includes both the table and relation definitions:

```typescript
// AUTO-GENERATED
import { relations } from 'drizzle-orm';

export const pets = pgTable('pets', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: varchar('name', { length: 100 }).notNull(),
  ownerId: uuid('owner_id').references(() => owners.id),
  // ...
});

export const owners = pgTable('owners', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: varchar('name', { length: 100 }).notNull(),
  email: varchar('email', { length: 255 }).notNull(),
});

export const petsRelations = relations(pets, ({ one }) => ({
  owner: one(owners, { fields: [pets.ownerId], references: [owners.id] }),
}));

export const ownersRelations = relations(owners, ({ many }) => ({
  pets: many(pets),
}));
```

## Value Objects in the Database

Show how Triad value objects map to database columns:

```typescript
const Money = t.value('Money', {
  amount: t.float64().min(0).doc('Monetary amount'),
  currency: t.enum('USD', 'CAD', 'EUR').doc('Currency code'),
});

// When a model uses a value object, it expands to individual columns:
const Adoption = t.model('Adoption', {
  id: t.string().format('uuid').identity().storage({ defaultRandom: true }),
  petId: t.string().format('uuid').storage({ references: 'pets.id' }),
  adopterId: t.string().format('uuid').storage({ references: 'owners.id' }),
  fee: Money,   // Value object — expands to fee_amount + fee_currency columns
  completedAt: t.datetime().storage({ defaultNow: true }),
})
.storage({
  tableName: 'adoptions',
  // Override how Money value object maps to columns:
  valueMapping: {
    fee: {
      amount: { column: 'fee_cents', transform: 'centsFromDollars' },  // Store as integer cents
      currency: { column: 'fee_currency' },
    },
  },
});
```

## Migrations Workflow

```bash
# 1. Define or update your Triad models with .storage() hints
# 2. Generate the Drizzle schema
triad db --output ./src/db/schema.ts

# 3. Generate SQL migration from schema diff
npx drizzle-kit generate

# 4. Review the generated migration SQL
cat drizzle/migrations/0001_*.sql

# 5. Apply migration
npx drizzle-kit migrate
```

Key points:
- Triad generates the Drizzle TypeScript schema
- Drizzle Kit generates SQL migrations from schema diffs
- You review and apply migrations normally
- The chain is: Triad model → Drizzle schema → SQL migration → Database

## Testing — Behavior Setup with Drizzle

```typescript
// test-setup.ts
import { drizzle } from 'drizzle-orm/node-postgres';
import { Pool } from 'pg';
import { migrate } from 'drizzle-orm/node-postgres/migrator';
import * as schema from './db/schema';
import { PetRepository } from './repositories/pet';

export async function createTestServices() {
  const pool = new Pool({ connectionString: process.env.TEST_DATABASE_URL });
  const db = drizzle(pool, { schema });

  await migrate(db, { migrationsFolder: './drizzle/migrations' });

  return {
    petRepo: new PetRepository(db),
    async cleanup() {
      // Truncate all tables between tests
      await db.delete(schema.pets);
      await db.delete(schema.adoptions);
    },
    async disconnect() {
      await pool.end();
    },
  };
}
```
